Angular Signals – Demo per principianti

1 · signal()

Un signal è un contenitore reattivo. Quando il suo valore cambia, Angular aggiorna la vista in automatico.

contatore = {{ contatore() }}

Nel componente (.ts): 
contatore = signal(0);          // crea il signal

contatore()                     // leggi il valore
contatore.set(5)                // imposta un valore fisso
contatore.update(v => v + 1)   // aggiorna in base al valore precedente

2 · computed()

Un computed è un valore derivato da uno o più signal. Si ricalcola automaticamente ogni volta che i signal da cui dipende cambiano.

contatore() {{ contatore() }}
doppio() {{ doppio() }}
saluto() {{ saluto() }}
Nel componente (.ts): 
nome    = signal('Marco');
doppio  = computed(() => this.contatore() * 2);
saluto  = computed(() => 'Ciao, ' + this.nome() + '!');

3 · effect()

Un effect è una funzione che viene eseguita ogni volta che i signal che legge cambiano. Utile per logging, localStorage, chiamate esterne.

Log (ultimi 5 cambiamenti):

@for (riga of logRighe; track riga) {
{{ riga }}
}
Nel componente (.ts): 
private _log = effect(() => {
  // viene riletto ogni volta che contatore o nome cambiano
  console.log('contatore =', this.contatore(), 'nome =', this.nome());
});

Riepilogo

API Cosa fa Si aggiorna
signal() Contiene un valore scrivibile Manualmente (.set / .update)
computed() Valore derivato (sola lettura) Automaticamente
effect() Effetto collaterale Automaticamente
